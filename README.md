# AppBlocker — Блокировщик приложений и сайтов

**AppBlocker** — защищённый Windows-блокировщик сайтов и приложений. Помогает удерживать концентрацию, запрещая доступ к отвлекающим сайтам и приложениям (играм, мессенджерам) на системном уровне. 

AppBlocker использует фоновую службу с правами SYSTEM, расширение для браузера, и систему межпроцессного взаимодействия (Native Messaging), что делает обход блокировок крайне сложным.

## Ключевые возможности

*   **Гибкий таймер сессий:** Быстрый запуск сессий фокусировки (например, 25, 60, 120 минут) или настройка кастомного времени.
*   **Глубокая блокировка процессов:** Постоянный мониторинг и мгновенное завершение запрещенных программ на системном уровне.
*   **Блокировка веб-сайтов:** Жесткая блокировка отвлекающих сайтов через редактирование файла `hosts` и специальное браузерное расширение (Chrome/Edge).
*   **Строгая защита от отключения (Friction):**
    *   Защита интерфейса паролем или PIN-кодом.
    *   Опциональное требование решить сложную математическую задачу перед досрочной остановкой сессии блокировки.
    *   Защита системной службы (`AppBlocker.Service`) от остановки обычным пользователем через модификацию дескрипторов безопасности (SDDL).
*   **Автозагрузка:** Автоматический старт вместе с Windows и работа в фоновом режиме в системном трее.

## Архитектура проекта

Проект разделен на несколько независимых модулей для обеспечения максимальной безопасности:

1.  **`AppBlocker.UI` (WPF, .NET 8):** Пользовательский интерфейс, работающий в пространстве пользователя. Отправляет команды фоновой службе через именованные каналы (Named Pipes).
2.  **`AppBlocker.Service` (Windows Service, .NET 8):** Фоновая служба, работающая с правами `SYSTEM`. Отвечает за мониторинг процессов, модификацию файла `hosts` и соблюдение времени сессии.
3.  **`AppBlocker.Watchdog` (.NET 8):** Сторожевой процесс, который следит за тем, чтобы служба AppBlocker.Service всегда работала (перезапускает её в случае падения или попытки убийства через диспетчер задач).
4.  **`AppBlocker.Bridge` (Console App, .NET 8):** Native Messaging Host, который служит мостом между браузерным расширением и `AppBlocker.Service`.
5.  **`AppBlocker.Extension` (JavaScript):** Расширение для Chromium-браузеров (устанавливается принудительно через Windows Registry Policy), перехватывающее запросы к сайтам из черного списка на лету.

## Установка

1.  Перейдите на страницу **[Releases](https://github.com/timursarsembai/AppBlocker/releases)**.
2.  Скачайте последнюю версию инсталлятора `AppBlockerSetup.exe`.
3.  Запустите установку от имени Администратора.
4.  Инсталлятор автоматически установит все службы, пропишет необходимые политики для браузеров и запустит приложение.

*(Для корректной работы расширения может потребоваться полностью перезапустить браузер).*

## Сборка из исходников

Для самостоятельной сборки вам понадобятся:
*   Visual Studio 2022 (с поддержкой разработки для .NET 8 и WPF).
*   .NET 8.0 SDK.
*   [Inno Setup 6](https://jrsoftware.org/isinfo.php) (для сборки инсталлятора).

### Шаги:
1. Клонируйте репозиторий.
2. Откройте `AppBlocker.sln` в Visual Studio и соберите решение в режиме `Release`.
3. Запустите скрипт сборки расширения (опционально, если меняли расширение).
4. Откройте `installer.iss` в Inno Setup и нажмите "Compile" — инсталлятор появится в папке `Output`.
